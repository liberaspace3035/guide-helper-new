# 動作確認後のアクションアイテム

## 必須確認項目

### 1. 環境設定
- [ ] `.env`ファイルの作成と設定
- [ ] データベース接続設定
- [ ] `APP_KEY`の生成（`php artisan key:generate`）
- [ ] `JWT_SECRET`の生成（`php artisan jwt:secret`）

### 2. データベース
- [ ] マイグレーション実行（`php artisan migrate`）
- [ ] 既存データがある場合はバックアップ
- [ ] マイグレーションと既存スキーマの整合性確認

### 3. 動作確認
- [ ] ホームページが表示される
- [ ] ログインページが表示される
- [ ] 登録ページが表示される
- [ ] ログイン機能が動作する
- [ ] 登録機能が動作する
- [ ] ダッシュボードが表示される（認証後）

### 4. API動作確認
- [ ] `POST /api/auth/register` が動作する
- [ ] `POST /api/auth/login` が動作する
- [ ] `GET /api/auth/user` が動作する（JWT認証）
- [ ] 各APIエンドポイントが正しく動作する

### 5. 認証・認可
- [ ] Web認証（セッション）が動作する
- [ ] API認証（JWT）が動作する
- [ ] ロールベースのアクセス制御が動作する
- [ ] 未認証ユーザーが保護されたページにアクセスできない

## 推奨確認項目

### UI/UX
- [ ] 既存ReactアプリとUIが一致している
- [ ] CSSが正しく読み込まれている
- [ ] 画像が正しく表示されている

- [ ] レスポンシブデザインが適切に動作している

### 機能
- [ ] 依頼作成が動作する
- [ ] 依頼一覧が表示される
- [ ] マッチング機能が動作する
- [ ] チャット機能が動作する
- [ ] 報告書機能が動作する

## 既知の問題と対処

### 1. APIルートのミドルウェア
- APIルートで`role`ミドルウェアを使用する場合、`auth:api`と組み合わせて使用
- `RoleMiddleware`はWebとAPIの両方で動作するように実装済み

### 2. 認証ガード
- Webルート: `auth`ミドルウェア（デフォルトは`web`ガード）
- APIルート: `auth:api`ミドルウェア（`jwt`ガード）

### 3. CORS設定
- APIエンドポイントへのアクセスでCORSエラーが発生する可能性
- `config/cors.php`の設定を確認
